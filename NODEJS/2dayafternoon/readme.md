클라이언트/서버 통신
-통신 프로토콜
    대표적인 통신 프로토콜로는 IBM의 폐쇄형 망 구조인 SNA와 개방형 망 구조인 TCP/IP가 있습니다. TCP/IP 응용 계층에 적용 확장된 프로토콜로는 전자 우편 서비스를 위한 SMTP, 파일 전송 서비스를 위한 FTP, 망 관리 서비스를 위한 SNMP, 그리고 우리가 주로 다루게 될 웹 서비스를 위한 HTTP 등이 있습니다.

-클라이언트/서버 통신 방식
    -Polling 방식(Ajax): 클라이언트가 서버에 주기적으로 요청 후 응답을 받는 방식
    단점: 트래픽 낭비, 실시간성 X
    장점: 쉬운 구현

    -Long Poll(Ajax) 방식: Client가 서버에 대한 요청을 유지하여 반복적인 요청을 없애고 유효한 이벤트가 발생하면 응답을 해주는 방식, 즉 접속을 오래 유지

    -WebSockt 방식: Client와 서버가 연결된 후부터는 HTTP 요청/응답에 관계 없이 서버와 양방향 통신이 가능

-Socket.io
    node.js 기반으로 만들어진 기술로, 자바스크립트로만 구현되어있다.
    웹 브라우저와 웹 서버의 종류와 버전을 파악하여 가장 적합한 기술을 선택하여사용한다.

    HTML5 웹소켓은 브라우저별로 지원하는 웹소켓 버전이 다르며 오래된 브라우저의 경우 아예 지원하지 않습니다. 따라서 자바스크립트를 이용하여 브라우저에 상관없이 실시간 웹을 구현할 수 있는 Socket.io를 좀 더 많이 사용하고 있습니다.
